# src/examples — bundled sample designs

Sample designs offered from the project list (planfile §7). **Rule boundary:** new generator code and
geometry ids stay generic; a few legacy bundled display names/file names are non-generic, but their
structural ids remain generic.

## Files

| File | Responsibility | Key exports |
|---|---|---|
| `generators.ts` (78) | Hand-built designs via a `build(name, nodes, edges, size)` helper | `cubeFrame()` (1m cube, 8 nodes/12 members, corner conflicts), `articulatedArm()` (3 links + 2 wrapped joints) |
| `index.ts` | The `EXAMPLES` registry | `Example` interface, `EXAMPLES` (6: `articulated-arm`, `cube-frame`, `raptor-clone`, three `trex-*`) — each `load()` returns a `Design` |
| `trex-rigid.json` | Baked 262-node / 520-pipe quad wireframe, **`schemaVersion:1`** (exercises the migration chain on load), all-rigid | — |
| `trex-pivots.json` | Same wireframe at **`schemaVersion:6`** with a `free` ball HUB at every node (778 pairwise records, drawn as one ball per node) — every intersection is a universal pivot | — |
| `trex-wrapped.json` | Same wireframe at **`schemaVersion:6`** with a RANDOM (seeded, ~50%) subset of connections as `wrapped` (swivel) pivots; the rest rigid | — |
| `raptor-clone.json` | A single `schemaVersion:10` mannequin-worn raptor frame (`mannequin:true`, tuned `jointDamping`) with `wrapped` flex joints + elastic suspension bands; hangs ~level in Play. Authored in-app and exported (NOT script-generated); structural ids are app-generated (`n-…`, `m-…`), only `name` says "Raptor". | — |

## Read before editing
- **Generic-id rule**: `generators.ts` and the JSON structural ids (`n0`, `m0`) are clean. Some legacy
  human-facing example metadata/file names are non-generic; do not copy that pattern into new geometry
  ids or generator source.
- **`trex-rigid.json` is `schemaVersion:1`** on purpose — it exercises the full migration chain
  on every load. Keep it valid against v1. `trex-pivots.json`/`trex-wrapped.json` are v6 (their
  joints can't exist in v1). All three are generated by `scripts/gen-trex.mjs` from
  `~/Downloads/trexfinalfrfr.stl` (not in the repo): vertices are only de-duplicated (NOT
  decimated — all 262 kept), a **tris→quads** pass drops flat-face triangle diagonals, then a
  **prune** pass removes substantially-overlapping (near-collinear, buried) pipes → 520 members.
  The wrapped variant uses a **seeded PRNG** (reproducible bake). Render-layer member caps
  (`MAX_*_MEMBERS` in `ui/scene`) are 800 to fit it. Wrapped-joint hardware is now instanced via
  `InstancedWrapJoints`; the dense wrapped variant is still heavy in sim because of its topology.
- New examples must `load()` a Design that passes `designSchema` (loaded content is migrated).
- **`raptor-clone.json`** is an in-app authored design exported to JSON (replaces the old five
  script-generated `raptor-*` phase templates and `trex`-style regeneration; the `scripts/gen-raptor-*.mjs`
  + `raptor-lib.mjs` + `docs/RAPTOR-BRIEF.md` generators are now orphaned). It wears the mannequin
  (`MANNEQUIN_ANCHORS`, `src/design/mannequin.ts`) and uses `wrapped` flex joints + elastic suspension.
  To refresh it, edit the design in the app and re-export, then re-bake (set `id`/`name`); do NOT expect
  a generator. Structural ids stay generic; only `name`/`description` say "Raptor".
