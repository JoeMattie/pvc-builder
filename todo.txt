Todos:

Bugs:
    -When length lock is enabled, free pivots are behaving like constrained ones: they are locked to an axis instead of being able to pivot freely. Fix this.
    - Sometimes, I end up with two overlapping pivots at the same union point. We should resolve these automatically. I think this happens when the mouse right click happens on the flat end of a pipe, so it adds another pivot even though there was already one serving that joint. Look at bug.pvc.json in my downloads folder for an example
    - Right click menu should only open up the join selector when right clicking on a join, not on a pipe or a multi selection. Right clicking on a pipe or multi selection should allow the pipe (and any unions) to be switched between 1/2 and 3/4.
    - Ctrl + drag on an end connection should break the union as long as a pipe was selected first to distinguish which end we are moving, allowing the end to move ALONG the pipe under the cursor (or freely if no pipe under the cursor). If an end connection is grabbed without first selecting a pipe, Ctrl+drag should do what it currently does (because we don't know which end to move and which to leave alone, so we move both of them). Clicking a pipe first, then Ctrl dragging on it's end should move that pipe's end and leave the other in place like I described above.

General:
    - Delete, move, and rotate should all work on multiple objects when selected.
    - A units pill/menu. Support mm, cm, decimal inch, fractional inch. This should only affect how they are displayed, not stored internally. Default should be decimal inches, rounded to something sane
    - The blue ball (used in drawing mode and end selection mode) should be slightly smaller, and partially transparent
    - Let's restrict it so nothing can be drawn or created below the ground plane.
    - Give the infinite ground plane some subtle color or at least make it a slightly different shade than the sky
    - Hotkeys should be shown on the screen on the tool they corrospond to
    - In addition to the current click to snap to camera views, we should have several isometric defaults there as well
    - Going back to the main menu and opening a document or creating a new one should reset camera and button states in the app to what is stored in the document, not carry over from the previous document. These states should be stored in the document.
    - Clicking and dragging with the left mouse on empty canvas should automatically switch to select mode unless we are currently in a drawing mode.
    - Double-clicking a pipe in select mode should start drawing a new pipe starting with the point that was double clicked on.


Joints:
    - When switching which pipe wraps which, keep the joint selected. A small switch gizmo should also appear floating next to the joint when selected.
    - When selecting a joint, don't select a pipe as well, just the joint, and highlight it.
    - We should add a "manufactured joint" to the right click menu, which will then replace the joint with a manufactured one and perform a bend on the pipe to make the end approach angle match the closest manufactured joint available.

Rendering:
    - Is it possible to use a boolean subtraction for drawing pipe interiors instead of a a black circle placed at the end? Discuss caveats with me before implementing.

Drawing mode:
    - When drawing pipes in click - move - click mode, typing after the 1st click should type into the length pill. Press enter to complete the pipe (create the second end) at the typed distance. If no units are supplied, use the default for the file. Support conversions. I should be able to type lengths like: 10mm, 10 mm, 10", 10 ", 10 in, 10in, 10ft, 1/2in, 1.5in, 1.5mm, etc.
    - When snapping is occuring (eg: the cursor is snapped to end of a pipe) we should indicate this with a small dot at the snapping point and a small text pill like "End". If we are snapping to another pipe, draw an outline around the pipe/pipes we are snapping TO
    - Snapping while in draw mode should automatically try to snap the ends of any pipe under the cursor, or along the length of that pipe. I want to be able to draw a pipe, change view, and then draw another pipe starting somewhere else or starting on another pipe and easily snap it to an existing pipe.

Move:
    When moving in the Y axis, show distance from ground

New:
    - A tapemeasure tool. Same ergonomics as the draw tool but only shows a measurement. ends of measurements should be able to snap to ends of objects as well as anywhere along a pipe. These are persistent, not ephemeral, and should be able to be selected and deleted. Immediately after placing a measurement, a third click (or second if click+drag was used to place the measurement) should allow the actual visualization to be placed, offset perpendicular to the axis of the measurement, at a distance.
    - A draw-on-plane tool: user selects it (hotkey F) and temporary square semi-transparent plane is shown under the mouse cursor. The cursor should snap to endpoints in the model. Clicking once (or starting a click+drag) sets the plane point. Mouse moves in a direction to set the plane angle, second click (or release in click+drag) enters the mode. Plane angles supported (snapped to) should be cardinal directions and relative-cardinal directions to pipe or pipes the point clicked belong to. When entering the mode, switch to isometric and zoom/rotate the camera to face that plane. Drawing in plane mode is constrained to that plane. Exit with escape or by selecting another tool, which will return the camera to where it was before entering the mode.
    - Create a new Bend tool (the current one will be renamed, see below). The new Bend tool will be able to bend existing pipes. when bending, the length should be able to be locked or allowed to grow/shrink as appropriate. It should have a toggle to lock end point angles. If that toggle is enabled, bends can only occur starting a short distance from the end point, so that a smooth bend can be achieved. Limit bend angles to what is feasable with PVC pipe.

BOM / Cutlist:
    - The Cutlist should take into account that we will be fabricating wrapped unions, and that wraps will take several inches of extra material to accomodate the squish, wrap around the pipe, return, unsquish, and have enough extra length for a bolt to be driven through the two pipes. Figure this out with math an then make it a bit longer - we can always cut the pipes shorter if they end up too long. Indicate on those entries what the actual section length is in a format like 18in + 8in = 26in so they can be written on the pipes when cutting
    - Similarly, pipes that receive a wrap at their endpoint should be extended by 1 inch plus 1 radius to accomodate for an end cap. This should also be drawn in the editor UI as a ghost (don't change the real geometry, just render it visually)
    - Perform splitting for manufactured unions when generating the cutlist.
